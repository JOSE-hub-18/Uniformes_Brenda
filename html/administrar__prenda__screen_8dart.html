Sincroniza [cantidadActual] con el stock registrado en [InventarioProvider].

◆ _guardarCambios()

Future< void > _guardarCambios ( )

Persiste el precio editado en la base de datos.

Regla de negocio: el precio debe ser un valor numérico válido. Si [inventario] es null o el precio no es parseable, la operación se cancela. Al completarse, cierra la pantalla devolviendo true al caller.

◆ build()

override Widget build ( BuildContext context)

Construye la interfaz de la pantalla.

Muestra un indicador de carga mientras [cargando] es true. Si [inventario] es null tras la carga, muestra un mensaje de error. En caso exitoso, renderiza los datos de la prenda, el campo de precio, el acceso a [AdministrarCantidadScreen], y los botones de guardar y eliminar.
