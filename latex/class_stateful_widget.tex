\doxysubsection{Descripción detallada}
Pantalla para modificar la cantidad de unidades de un inventario específico. 

Pantalla que lista las ventas registradas y permite filtrarlas por año y mes.

Pantalla para revisar los detalles de una venta.

Pantalla encargada de mostrar el detalle de un pedido pendiente, permitiendo registrar unidades mediante códigos QR, eliminar prendas del pedido y completar el pedido cuando todas las unidades han sido registradas.

Pantalla para registrar una nueva prenda en el inventario.

Pantalla que implementa un escáner de códigos QR usando {\ttfamily mobile\+\_\+scanner}.

Pantalla que muestra la lista de pedidos pendientes.

Pantalla para crear una nueva orden o venta.

Pantalla de inicio de sesión con animaciones de entrada y validación.

Pantalla de confirmación visual que indica el éxito de una operación.

Pantalla para agregar prendas a un pedido existente.

Pantalla para visualizar y modificar los datos de una prenda de inventario.

Expone dos operaciones\+: agregar unidades mediante impresión de QRs, y restar unidades mediante escaneo de QRs físicos. Requiere \mbox{[}id\+Inventario\mbox{]} para identificar el inventario sobre el que opera.

Permite editar el precio, consultar la cantidad disponible, navegar a \mbox{[}Administrar\+Cantidad\+Screen\mbox{]} para modificar unidades, y eliminar el registro del inventario.

Permite filtrar el inventario disponible por escuela y tipo de prenda, y agregar unidades al pedido activo. Nota\+: los datos de escuelas, prendas e inventario son estáticos en esta versión.

Componente de tipo \mbox{[}\doxylink{class_stateful_widget}{Stateful\+Widget}\mbox{]} diseñado para notificar de forma efímera al usuario que una transacción o modificación de datos ha culminado exitosamente.


\begin{DoxyItemize}
\item Implementa un formulario simple con campos de usuario y contraseña.
\item Usa \mbox{[}Auth\+Provider\mbox{]} para realizar la autenticación y mostrar estado de carga/errores.
\item Incluye animaciones de entrada (fade + slide) controladas por un \mbox{[}Animation\+Controller\mbox{]}.
\item Permite seleccionar escuela, prenda y talla desde el inventario.
\item Soporta agregar ítems manualmente o mediante escaneo de QR.
\item Muestra las prendas agregadas agrupadas por inventario y permite confirmar la orden.
\item Componente de tipo \mbox{[}\doxylink{class_stateful_widget}{Stateful\+Widget}\mbox{]} que consulta al proveedor \mbox{[}Pedidos\+Pendientes\+Provider\mbox{]} para obtener los pedidos pendientes.
\item Permite crear un nuevo pedido y revisar pedidos existentes.
\item La recarga de datos se realiza en {\ttfamily init\+State} mediante el provider.
\item Recibe un callback {\ttfamily on\+Scan} que procesa el valor del QR y devuelve un {\ttfamily Scan\+Feedback}.
\item {\ttfamily mostrar\+Mensaje} controla si se muestra un mensaje visual tras el escaneo.
\item Presenta un formulario con campos\+: tipo de prenda, precio, talla y escuela.
\item Utiliza un caso de uso \mbox{[}\doxylink{class_registrar_inventario_use_case}{Registrar\+Inventario\+Use\+Case}\mbox{]} para ejecutar la operación de registro y muestra retroalimentación al usuario mediante Snack\+Bars.
\item Muestra los ítems vendidos asociados a una venta específica.
\item Permite devolver prendas (reactivar unidad, eliminar detalle de venta), reimprimir QRs y actualizar el inventario y listados relacionados.
\item Usa \mbox{[}Ventas\+Provider\mbox{]} para obtener y observar el listado de ventas.
\item Permite navegar a la vista de revisión de una venta individual. 
\end{DoxyItemize}
