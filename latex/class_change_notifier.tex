\doxysubsection{Descripción detallada}
Proveedor de estado para la autenticación de usuarios. 

Provider de estado para la gestión del historial de ventas.

Provider de estado para la pantalla de reportes de ventas.

Provider de estado para el flujo de registro de inventario.

Provider de estado para la gestión de unidades obtenidas mediante escaneo de QR.

Provider de estado para la gestión de unidades con QR pendiente de impresión.

Provider de estado para la gestión de impresión de QRs via Bluetooth.

Provider de estado para la gestión de pedidos pendientes.

Provider de estado para el flujo de creación de una nueva orden.

Provider de estado para la pantalla principal de la aplicación.

Provider de estado para la gestión de alertas de stock del inventario.

Proveedor de estado para el módulo de inventario.

Gestiona la carga y filtrado de inventario por escuela y prenda, así como el estado de selección activa del usuario. Extiende \mbox{[}\doxylink{class_change_notifier}{Change\+Notifier}\mbox{]} para notificar cambios a los widgets suscritos.

Consume \mbox{[}Alertas\+Stock\+Use\+Case\mbox{]} para obtener los registros de inventario en estado agotado o crítico, y notifica a los widgets suscritos ante cualquier cambio de estado.

Gestiona el nombre del usuario autenticado y centraliza la navegación hacia las secciones principales del sistema.

Coordina la interacción entre \mbox{[}\doxylink{class_nueva_orden_use_case}{Nueva\+Orden\+Use\+Case}\mbox{]}, \mbox{[}\doxylink{class_inventario_repository}{Inventario\+Repository}\mbox{]} y \mbox{[}Alertas\+Provider\mbox{]} para gestionar los items de la orden, ya sea por escaneo de QR (venta) o por selección manual (pedido).

Coordina \mbox{[}\doxylink{class_pedido_pendiente_use_case}{Pedido\+Pendiente\+Use\+Case}\mbox{]} para cargar, registrar, desregistrar y completar pedidos, notificando a los widgets suscritos ante cada cambio.

Coordina \mbox{[}\doxylink{class_print_use_case}{Print\+Use\+Case}\mbox{]} con la solicitud de permisos de Bluetooth y la actualización de \mbox{[}Alertas\+Provider\mbox{]} tras cada operación de impresión.

Consume \mbox{[}\doxylink{class_unidad_repository}{Unidad\+Repository}\mbox{]} para obtener las unidades que han sido registradas en el sistema pero cuyos QRs aún no han sido impresos físicamente.

Consume \mbox{[}\doxylink{class_qr_use_case}{Qr\+Use\+Case}\mbox{]} para resolver los códigos QR escaneados a instancias de \mbox{[}Unidad\mbox{]}, manteniendo una lista acumulada de unidades procesadas en la sesión.

Gestiona la carga de catálogos de prendas, tallas y escuelas, y expone la operación de agregar nuevas escuelas al sistema.

Gestiona los filtros de año, mes y escuela, y consume \mbox{[}\doxylink{class_reporte_repository}{Reporte\+Repository}\mbox{]} para obtener los indicadores de ventas y los rankings de prendas y escuelas.

Consume \mbox{[}\doxylink{class_ventas_use_case}{Ventas\+Use\+Case}\mbox{]} para obtener el listado de ventas y sus detalles, y coordina con \mbox{[}Alertas\+Provider\mbox{]} para refrescar alertas de stock ante operaciones que modifiquen el inventario.
